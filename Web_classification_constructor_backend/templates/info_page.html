Инструкция по работе с конструктором

Данный проект представляет собой инструмент для бинарной классификации. Настоятельно советуем Вам прочитать инструкцию до того, как что-то делать.

У классификатора есть два основных режима:
  • Режим создания модели с нуля с предсказанием и полным отчетом
  • Режим с использованием предобученный модели и последующим предсказанием

Режим создания модели с нуля

Для данного этапа Вы должны нажать Создать модель . В данном режиме Вам необходимо самостоятельно задавать параметры для препроцессинга данных, базовых алгоритмах. Все появляющиеся окна обязательны для заполнения(???). Более подробно о параметрах будет сказано ниже.

После заполнения данных (первые 2 страницы) Вам надо загрузить файлы с обучающей и тестовой выборкой. К данным файлам выдвигаются важные требования, о которых будет сказано здесь.

Далее Вы нажимаете кнопку Upload , файлы проверяются, если всё хорошо, то надо нажать кнопку Далее , после чего на следуюшей странице млжно нажать Begin work и начнутся непосредственные вычисления и генерация архива с нужными файлами.Этапы данного процесса выводятся на экран, после завершения всех этапов появляется возможность нажать на кнопку Скачать отчет , при нажатии на которую будет скачан файл results.zip

Также на каждом из этапов можно вернуться к главной странице нажатием кнопки В начало .

Описание параметров всех используемых алгоритмов обучения

Перед описанием стоит отметить, что заполнение параметров проходит в два этапа и от выбранных параметров из первого этапа будет зависеть выбор параметров на втором. Поэтому описание будет строиться так: Описывается конкретный параметр первого этапа и в этом описании указываются соответствующие ему параметры второго с их объяснением.

  • Метод заполнения пробелов
    Для данного метода существуют 3 альтернативы:
    • 1. HardRemoval: Удаление всех строк, в которых есть хотя бы один пропуск. При таком выборе на втором этапе ничего заполнять не нужно
    • 2. InsertMeanMode: Заполнение всех пропусков средним или модой по конкретному признаку. Требует на втором этапе параметр threshold - пороговое количество уникальных значений признака (натуральное число). При его превышении вычисляется среднее, иначе - мода.
    • 3. LinearImputer: Заполнение пропусков путем их предсказывания с помощью линейной регрессии. Не требует особых параметров.
  • Метод удаления выбросов
    Для данного метода существуют следующие варианты:
    • 1. ThreeSigma. Не требует дополнительных параметров
    • 2. Grubbs:
      • Требует определить следующие параметры
      • Alpha: - уровень значимости гипотезы (число от 0 до 1)
    • 3. Interquartile:
      • Требует определить следующие параметры
      • Low quant - нижний квартиль (число от 0 до 1 ). Лучше брать 0.25
      • Up quant - верхний квартиль (число от 0 до 1). Лучше брать 0.75
      • Coef - коэффициент для растета расстояния
    • 4. IsolationForest:
      • Требует определить следующие параметры
      • N estimators - количество деревьев (рекомендуется 100)
      • Contamination - доля выбросов, которая будет отсеяна (числа от 0 до 1)
    • 5. Elliptic:
      • Требует определить следующие параметры
      • Contamination - доля выбросов, которая будет отсеяна (числа от 0 до 1)
    • 6. SVM:
      • Требует определить следующие параметры
      • Iters - количество итераций алгоритма
    • 7. Approximate:
      • Требует определить следующие параметры
      • Deviation - разница ранга матриц (целое число)
    • 8. LocalFactor:
      • Требует определить следующие параметры
      • Neigh - количество соседей (рекомендуется 20)
      • Contamination - доля выбросов, которая будет отсеяна (числа от 0 до 1)
  • Этап отбора признаков
    Для данного метода существуют следующие варианты:
    • 1. VarianceThreshold:
      • Требует определить следующие параметры
      • Threshold - Пороговая дисперсия. Все признаки с дисперсией ниже пороговой удаляются
    • 2. SelectKBest:
      • Требует определить следующие параметры
      • K - количество отбираемых признаков
    • 3. SelectPercentile:
      • Требует определить следующие параметры
      • Percentile - Процент (от 0 до 100) отбираемых признаков
    • 4. SelectFpr:
      • Требует определить следующие параметры
      • Alpha - уровень значимости (от 0 до 1)
    • 5. SelectFdr:
      • Требует определить следующие параметры
      • Alpha - уровень значимости
    • 6. SelectFwe
      • Требует определить следующие параметры
      • Alpha - уровень значимости
    • 7. GenericUnivariateSelect:
      • Требует определить следующие параметры
      • Mode - режим (объединяет предыдущие варианты)
      • Param - соттветствующее число
    • 8. SelectFromModel:
      • Требует определить следующие параметры
      • Threshold - весовой порог
      • Norm_order - хз, ставьте 1
      • Max_features - максимальное число оставляемых признаков, преодолевших порог
  • Метод композиции
    Для данного метода существуют следующие варианты:
    • 1. voting. Для данной композиции дополнительыне парамтеры не нужны
    • 2. adaboost:
      • Требует определить следующие параметры
      • N estimators - количество базовых алгоритмов. Примерный порядок чисел —десятки-сотни. Рекомендуется брать 50
      • Learning rate - скорость обучения. Рекомендается брать 1
      • Algorithm - 'SAMME' (дискретный алгоритм) или 'SAMME.R' (вещественный алгоритм бустинга). Луше брать 'SAMME.R' (он быстрее)
    • 3. stacking:
      • Требует определить следующие параметры
      • Stack method - имеет 4 варианта: ‘auto’ (по умолчанию), ‘predict_proba’, ‘decision_function’, ‘predict’. Означает методы, которые будут вызываться у каждого частного алгоритма. ‘auto’ является универсальным вариантом, который объединяет ‘predict_proba’, ‘decision_function’, ‘predict’.
      • Cv - количество блоков для кросс-валидации финальной модели. Рекомендуется брать 5
  • Базовые алгоритмы
    К базовым алгоритмам относятся нейронные сети, деервья решений и логистическая регрессия. На первом этапе надо указать количество каждого базового алгоритма (либо просто тип в случае adaboost). Если данный базовый алгоритм не используется, нало указать 0 (в случае voting и stacking). Далее приводится описание алгоритов, их параметры и описание параметров
    • 1. Нейронные сети:
      • Требует определить следующие параметры
      • Activation - функция активации для скрытого слоя
      • Solver - алгоритм для оптимизации весов
      • Learning rate - стратегия обновления скорости обучения
      • Learning rate init - начальная скорость обучения
    • 2. Деревья решений:
      • Требует определить следующие параметры
      • Criterion - функция для измерения качества разбиения.
      • Max depth - максимальная глубина дерева
      • Min samples split - минимальное число объектов в листе, при котором его можно разбить еще на 2 листа
      • Min samples leaf - минимальное число объектов, которые в принципе могут быть в листе
    • 3. Логистическая регрессия:
      Требует определить следующие параметры
      • Solver - алгоритм для решения проблем оптимизации
      • Penalty- норма, используемая для штрафа
  • Test_ratio
    Это доля валидационной выборки, на основе которой будет смотреться качество и формироваться итоговый отчет. Вещественное число в интервале от 0 до 1 (???) Рекомендуемое значение - 0.25

Требования к загружаемым файлам

  • В режиме модели с нуля:

    Для файла обучающей выборки: В файле первый столбец — это идентификатор с названием столбца id. Далее идут столбцы-признаки в численном виде (названия столбцов могут быть любыми кроме id и target их уже использовали). Работа с категориальными признаками в данной программе не предусмотрена. Предварительная нормализация не обязательна - она выполняется внутри классификатора Далее должен быть столбец target с целевой переменной, т.е. в каждой строке указывается метка класса 0 или 1.

    Для файла тестовой выборки: В файле первый столбец — это идентификатор с названием столбца id. Далее идут столбцы-признаки в численном виде (названия столбцов могут быть любыми кроме id, так как его уже использовали)

  • В режиме готовой модели:

    Файл модели должен быть созданным в этом конструкторе !!!

    Требования к файлу тестовой выборки такие же, как в первом режиме

Содержимое отчета

В режиме модели с нуля окончательынм артефактом является архив results.zip, в который включаются:

  • Графики
  • Модели в формате pickle (сериализованный список, в котором первый элемент - сама модель, а второй - признаки, на которых эта модель строилась
  • Отчет в формате .docx
  • Csv-файл с проставленными метками и вероятностями отнесения к классу

В режиме загружаемой модели окончательным артефактом является только Csv-файл
